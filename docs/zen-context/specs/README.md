# Specs

Specs in this directory define behavior before implementation.

Use `../spec-template.md` for new specs.

Naming convention:

```text
NN-short-name.md
```

Examples:

- `01-raycast-command-exec-helper.md`
- `02-current-tab-context.md`
- `03-profile-selection.md`

Each spec should include:

- summary;
- goals and non-goals;
- proposed design;
- API or contract;
- security and permissions review;
- test plan;
- open questions.

Implementation should not start for non-trivial behavior changes until the relevant spec is accepted.
